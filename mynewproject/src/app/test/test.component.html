HTTP and Observables

Currently we are serving the data that is hardocded in the employee service. But in real world application we need to fetch the data from a web server or web api. Lets understand the mechanism of HTTP call and the observable that is returned by the HTTP call. So we have an application that runs on the browser and we have 2 components, em list and emp detail. Both these components make use of EmployeeService for thier data. But we want the data to be fetched from the server and for that we make a HTTP get request. The HTTP get request will hit a web server which will fetch a data from a DB and send it back as a http response. Response we get back from an HTTP call is an observable. Now the employeeService needs to cast the observable into an array of employees and return to the emp list and emp detail. We provide the data only to the components that have subscribed to the EmployeeService. After this it its upto the components what to do with the data. So the HTTP mechanism is just 2 steps, send the http request and receive and process HTTP response from the server.

Observables

  • A sequence of items that arrive asynchronously over time.
  • HTTP call is a single item and not a sequence of items
  • Single item is HTTP response that arrives asynchronously.

HTTP, Observables and RxJS

  • Make HTTP GET request from the EmployeeService

RxJS is an external library that enables us to work with observables in Angular applications.
